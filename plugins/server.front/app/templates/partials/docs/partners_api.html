OAuth.io Partners API Reference

The website REST API for partners

What is it?

Throught the OAuth.io Partners REST API, you will be able to create and manage users on OAuth.io, thus making you a partner of our service. We provide a set of endpoints that allow you to create, remove and get information about your OAuth.io users. Then, you can manage their apps, domains, keysets and receive the providers' configurations of their apps.

How to become a partner?

In order to do so, you will have to follow these simple steps:

- Create a standard user account on OAuth.io

- Take contact with our team and ask to become a partner

- Our team will choose with you a platform_name

- You will be granted administration right over every user you create using this platform_name to make request to our Partner API endpoints.

This API is not a required part of the authentication or authorization feature: You cannot use this API to auth to a provider or any related feature about their access tokens. To do so, use the Javascript API.

How to use OAuth.io endpoints?

Every request shoud be authorized with a bearer token.

You can receive it by using the /auth/token endpoint, and authorize your request by setting the http header Authorization: Bearer YOUR_TOKEN

If not, you will receive a 401 Unauthorized response.

Check the authentication endpoint for more informations!

Some common notes about the methods below:

Each described endpoints below are relative to https://oauth.io. Every endpoints beginning with /api always reply a response structure according to JSend: The field status can have the values success, fail, error, depending on the error type. Beside the status field, you can find the field message when an error is present, and data in case of success or when a parameter is invalid.

Every POST request should send JSON in its body with the HTTP header Content-Type set to application/json. Some endpoints can accept application/x-www-form-urlencoded, but we recommand to always use JSON.


The platform_name refers to you partner name id, given to you when you become a partner.

The user_email refers to the email of the user you are managing. You have to give a valid email to each user you create.

Authentication

HTTP POST /auth/token

Authenticate to oauth.io with OAuth 2 client credentials.
grant_type

Required

 string

Must be "client_credentials"

Authorization header

Required

 string

An HTTP Basic authentication, thus base64(login + ":" + password) with your login and password of your OAuth.io account.

Node Request example:

var b = new Buffer("example@mail.com:mypassword");
var authorization = b.toString('base64');

var apiAuthentication = {
    url: 'https://oauth.io/auth/token',
    method: 'POST',
    json: {
        grant_type: "client_credentials"
    },
    headers: {
        'User-Agent': 'request',
        'Authorization' : "Basic " + authorization
    }
};

request(apiAuthentication, function(error, response, body) {
    console.log("apiAuthentication", body);
    var access_token = body.access_token;
};

Response example:

{
    "access_token": "KZMx0AExV6scnfR3EqcFb8ck6jY",
    "token_type": "Bearer",
    "expires_in": 18000
}
