What is OAuth

OAuth is an open standard for authorization. OAuth provides a method for clients to access server resources on behalf of a resource owner (such as a different client or an end-user). It also provides a process for end-users to authorize third-party access to their server resources without sharing their credentials (typically, a username and password pair), using user-agent redirections.

Wikipedia, OAuth

In practice, influential web companies could not wait for the IETF to finish the OAuth specification (whichever the version). This resulted in a huge fragmentation not only regarding the official versions of the protobut also regarding implementations that simply are not compliant with any of the specifications. Hence developers pain.

Why do you store my private keys?

Currently, we need to store your API keys in order to authenticate/authorize your user. Think of OAuth.io as your OAuth backend. However, we are actively thinking on better ways to handle this in a privacy-friendly fashion.

How secure is it?

All requests are HTTPS, meaning that they are encrypted using TLS/SSL. This prevents anyone from hijacking a temporary but valid OAuth token.
