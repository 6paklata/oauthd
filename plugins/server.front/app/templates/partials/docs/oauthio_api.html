OAuth.io API Reference

The website REST API

This API is not a required part of the authentication or authorization feature: You cannot use this API to auth to a provider or any related feature about their access tokens. To do so, use the Javascript API

The REST API is only used to manage your actions on OAuth.io: you can manage your apps, domains, keysets, receive the providers' configuration or your account informations.

How to useOAuth.io endpoints

Each described endpoint below is relative to https://oauth.io.

Every request shoud be authorized with a bearer token. You can receive it by using the /auth/token endpoint, and authorize your request by setting the http header Authorization: Bearer YOUR_TOKEN

If not, you will receive a 401 Unauthorized response.

Some common notes about the methods below:

Every POST request’s body should be formatted in JSON, and have the header Content-Type set to application/json.

The response will contain the following fields :

- status, that can hold success, fail or error

- message, if an error occurred, with more details

- data, which contains the response

Authentication

HTTP POST /auth/token

Authenticate to oauth.io with OAuth 2 client credentials.
grant_type

Required

 string

Must be "client_credentials"

Authorization header

Required

 string

An HTTP Basic authentication, thus base64(login + ":" + password) with your login and password of your OAuth.io account.

Response example:

{
	"access_token": "KZMx0AExV6scnfR3EqcFb8ck6jY",
	"token_type": "Bearer",
	"expires_in": 18000
}

Proxy API

HTTP GETPOSTPUTDELETEPATCH /request/{provider}/{url}

provider

Required in url

 string

The provider's name (e.g. "facebook")

request_url

Required in url

 string

the api request's url, can be relative to the main api domain (e.g api.facebook.com) or absolute. This url must be urlencoded.

You must include the "oauthio" HTTP header inside your request, which is a application/x-www-form-urlencoded hash containing:

- k: the public OAuth.io key of your app. If the http header Origin or Referer is set, this will be checked against your app’s whitelist. If none are present, you must accept the “localhost” domain in your OAuthio’s app.

- oauth_token: OAuth1 public token.

- oauth_secret_token: OAuth1 secret token.

- access_token: OAuth2 token.

- oauthv (optional): when a provider supports both OAuth1 and OAuth2, this can be set to “1” or “2” to desambiguate the version of OAuth to use.


Example of Proxy API requests:

https://api.twitter.com/1.1/account/verify_credentials.json

Response example:

GET /request/twitter/%2F1.1%2Faccount%2Fverify_credentials.json HTTP/1.1
Host: oauth.io
oauthio: k=OAUTHIO_PUBLIC_KEY&oauthv=1&oauth_token=TWITTER_PUBLIC_TOKEN&oauth_token_secret=TWITTER_SECRET_TOKEN
Referer: https://whitelisted_domain_in_oauthio/
