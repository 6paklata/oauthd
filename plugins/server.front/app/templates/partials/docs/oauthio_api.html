OAuth.io API Reference

The website REST API

This API is not a required part of the authentication or authorization feature: You cannot use this API to auth to a provider or any related feature about their access tokens. To do so, use the Javascript API

The REST API is only used to manage your actions on OAuth.io: you can manage your apps, domains, keysets, receive the providers' configuration or your account informations.

OAuth endpoints

Some common notes about the methods below:

Each described endpoints below are relative to https://oauth.io. Every endpoints beginning with /api always reply a response structure according to JSend: The field status can have the values success, fail, error, depending on the error type. Beside the status field, you can find the field message when an error is present, and data in case of success or when a parameter is invalid.

Every POST request should send JSON in its body with the HTTP header Content-Type set to application/json. Some endpoints can accept application/x-www-form-urlencoded, but we recommand to always use JSON.

Finally, every request shoud be authorized with a bearer token (if not, you will receive a 401 Unauthorized response): You can receive it by using the /auth/token endpoint, and authorize your request by setting the http header Authorize: Bearer YOUR_TOKEN

Authentication

HTTP POST /auth/token

Authenticate to oauth.io with OAuth 2 client credentials.
grant_type

Required

 string

Must be "client_credentials"

Authorization header

Required

 string

An HTTP Basic authentication, thus base64(login + ":" + password) with your login and password of your OAuth.io account.

Response example:

{
	"access_token": "KZMx0AExV6scnfR3EqcFb8ck6jY",
	"token_type": "Bearer",
	"expires_in": 18000
}

Apps

HTTP POST /api/apps

Create an application.
name

Required

 string

The new name of the application, 3 to 50 chars.

domains

Required

 array of strings

Array of valid urls scheme/domain/port/path. You can choose the url detail level to use.

Response example:

{
	"status": "success",
	"data": {
		"name": "My test app",
		"key": "171oqMx5yYMDMS2HR7JQJMoBHfI"
	}
}
HTTP GET /api/apps/{key}

Get an app's details.
key (url) string

The app's public key

Response example:

{
	"status": "success",
	"data": {
		"name": "My renamed app",
		"key": "2-E1sctnaiCRjFlwOI8C3ptu2bU",
		"secret": "2ENMOzoIX69VIdMxnNDIDUEY1wg",
		"domains": [
			"https://example2.com:42/callback",
			"oauthio://example3.com",
			"example.com"
		],
		"keysets": [
			"facebook"
		]
	}
}

HTTP GET /api/apps/{key}/keysets/{provider}

Get a keyset for an app and a provider. Returns null if the keyset does not exists.
key (url) string

The app's public key

provider (url) string

The provider's name

Response example:

{
	"status": "success",
	"data": {
		"parameters": {
			"client_id": "987987987987987",
			"client_secret": "123c123b123e123f123c123bd123e123",
			"scope": [
				"email",
				"read_stream",
				"read_friendlists"
			]
		},
		"response_type": "code"
	}
}
HTTP POST /api/apps/{key}/keysets/{provider}

Add or update a keyset for an app and a provider. If the keyset already exists, it will be replaced.
key (url) string

The app's public key

provider (url) string

The provider's name

response_type

Required

 string

The response type can be "token" (client-side), "code" (server-side), or "both"

parameters

Required

 object

The parameters of the keyset, according to the provider's configuration.

Response example:

{
	"status": "success",
	"data": null
}
HTTP DELETE /api/apps/{key}/keysets/{provider}

Remove a keyset from an app by provider's name.
key (url) string

The app's public key

provider (url) string

The provider's name of the keyset to delete

Response example:

{
	"status": "success",
	"data": null
}

HTTP GET /api/providers/{provider}

Get a provider's configuration.
provider (url) string

The provider's name

extend

Required

 boolean

Set to "true" to extend the configuration with default parameters. "false" by default.

Response example:

{
    "status": "success",
    "data": {
        "url": "https://api.twitter.com/oauth",
        "oauth1": {
            "request_token": "/request_token",
            "authorize": "/authenticate",
            "access_token": "/access_token",
            "parameters": {
                "client_secret": "string",
                "client_id": "string"
            }
        },
        "href": {
            "keys": "https://dev.twitter.com/apps/new",
            "docs": "https://dev.twitter.com/docs",
            "apps": "https://dev.twitter.com/apps",
            "provider": "https://www.twitter.com/"
        },
        "name": "Twitter",
        "provider": "twitter"
    }
}
